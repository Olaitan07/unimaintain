UniMaintain API

Complete REST API for UniMaintain — a university maintenance request management system. All responses use a consistent envelope format. All protected endpoints require a Bearer JWT token in the Authorization header.

https://unimaintain.onrender.com
https://unimaintain-frontend.onrender.com
HTTPS · REST · JSON
JWT Bearer Token
Response Format

All responses follow a consistent envelope:

Success
{ "success": true, "data": { ... } }

Error
{ "success": false, "error": { "message": "...", "details": null } }

POST Create Request /api/requests
🔒 Requires Authorization: Bearer <token> · Roles: STUDENT, STAFF only

Submit a new maintenance request. Send as multipart/form-data to support optional image upload. The image field accepts PNG, JPG, JPEG, and WEBP files.

Body (form-data)
Response
FieldTypeRequiredDescription
titlestringRequiredShort title of the fault
descriptionstringRequiredDetailed description of the issue
categoryIdstringRequiredCUID of the category (from GET /categories)
locationstringRequiredWhere the fault is located
priorityenumOptionalLOW | MEDIUM | HIGH (default: MEDIUM)
imagefileOptionalPhoto of the fault (PNG/JPG/WEBP)
201 Created

PATCH Update Request Status /api/requests/:id
🔒 Requires Authorization: Bearer <token> · Roles: OFFICER (assigned requests only), ADMIN

Update the status of a maintenance request. Creates a StatusLog entry recording the actor, old status, new status, and optional comment.

Status flow: PENDING → ASSIGNED → IN_PROGRESS → COMPLETED

Body
Response
Content-Type: application/json
{
  "status":  "IN_PROGRESS",
  "comment": "Officer attended site, repair in progress"
}
FieldTypeRequiredDescription
statusenumRequiredPENDING | ASSIGNED | IN_PROGRESS | COMPLETED
commentstringOptionalNote recorded in the status log
200 OK 
{
  "success": true,
  "data": {
    "id":        "clz3aaa",
    "status":    "IN_PROGRESS",
    "updatedAt": "2026-06-21T14:00:00.000Z"
  }
}
Admin /api/admin
POST Assign Officer to Request /api/admin/requests/:id/assign
🔒 Requires Authorization: Bearer <token> · Role: ADMIN only

Assign a maintenance officer to a request. Creates an Assignment record, sets the request status to ASSIGNED, and sends the officer an in-app notification.

Body
Response
Content-Type: application/json
{
  "officerId": "clz1off001"
}
200 OK 
{
  "success": true,
  "data": {
    "id":          "clz3aaa",
    "status":      "ASSIGNED",
    "assignedTo":  { "id": "clz1off001", "name": "Olivia Officer" }
  }
}

Returns all notifications for the currently authenticated user, newest first. Notifications are created on officer assignment and status updates.

PATCH Mark Notification as Read /api/notifications/:id/read
🔒 Requires Authorization: Bearer <token>

Marks a single notification as read. Only the notification owner can mark their own notifications.

Response 200
{
  "success": true,
  "data": { "id": "clznot001", "read": true }
